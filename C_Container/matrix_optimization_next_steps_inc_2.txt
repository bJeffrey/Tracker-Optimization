motion/ folder contents

Files:
- track_batch_soa.hpp
  * TrackBatchSoA (SoA state + motion metadata)
- model_buckets.hpp
  * ModelBuckets (persistent model index lists, O(1) move via swap-remove)
- motion_models.hpp
  * IMotionModel interface + MotionModel_CA9 + MotionModel_CT_XY_OmegaZ
  * ModelSelector stub (single-best-model)
- scenario_loop.hpp
  * run_scenario_loop(...) with dt tick + scan tick callback

Integration checklist (minimal):
1) Add include/motion/* to your repo.
2) In src/main.cpp (or a new demo src), include:
     #include "motion/scenario_loop.hpp"
3) Create TrackBatchSoA tb; tb.resize(N); initialize state.
4) Create ModelBuckets buckets; buckets.init_from_track_batch(tb);
5) Create models:
     motion::MotionModel_CA9 ca;
     motion::MotionModel_CT_XY_OmegaZ ct;
     motion::ModelSelector selector;
6) Call run_scenario_loop(...) and in on_scan_tick:
     - build/update your rtree_sqlite from tb.pos_x/pos_y/pos_z (or tb.x/y/z)
     - query candidates using scan_volume

Later performance step:
- Replace scalar loops inside motion models with calls to your batch backends (MKL/Eigen/STD).
